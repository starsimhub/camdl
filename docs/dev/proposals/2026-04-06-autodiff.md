# Proposal: Source-to-Source Autodiff for camdl
**Motivation:** PGAS mixing is limited by 1D random-walk MH proposals on the
θ|X step. Each parameter moves ~0.3 units per sweep due to the sharp
complete-data posterior (46K Binomial terms constraining θ). HMC/NUTS with
gradients would propose all parameters jointly, traversing the posterior in
~10 leapfrog steps instead of zig-zagging for thousands of sweeps. Expected
speedup: 10–50× ESS per sweep.

**Approach:** Symbolic differentiation in the OCaml compiler. No runtime AD
machinery (no tapes, no dual numbers). The compiler emits derivative
expressions alongside the original expressions in the IR JSON. Rust
evaluates both using the existing `eval_expr` interpreter.

**Precedent:** Stan's compiler (stanc3) is written in OCaml and performs
a similar source-to-source transformation, though Stan's math library
also uses runtime reverse-mode AD. Our approach is simpler because the
camdl expression language is closed (no user-defined functions).

## Architecture

### Current pipeline (unchanged)

```
.camdl → OCaml parser → AST → IR (expr trees) → JSON → Rust deserializes → eval_expr interprets
```

### New pipeline (additions in bold)

```
.camdl → OCaml parser → AST → IR (expr trees)
                                  ↓
                          **differentiate(expr, param) → derivative expr**
                                  ↓
                         IR + derivative exprs → JSON → Rust deserializes
                                                          ↓
                                              eval_expr evaluates both
                                              value AND gradient
```

No changes to JSON serialization format, Rust deserialization, or `eval_expr`.
Derivative expressions are just more `expr` nodes in the IR.

## Part 1: OCaml Compiler — `differentiate`

### The `expr` type (from `ocaml/lib/ir/ir.ml`)

```ocaml
type bin_op = Add | Sub | Mul | Div | Pow | Mod | Min | Max | ...
type un_op = Neg | Exp | Log | Sqrt | Abs | Floor | Ceil

type expr =
  | Const  of float
  | Param  of string
  | Pop    of string          (* compartment population — constant in θ|X *)
  | PopSum of string list     (* sum of compartments — constant in θ|X *)
  | Time                      (* simulation time — constant in θ *)
  | BinOp  of bin_op_expr     (* { op; left; right } *)
  | UnOp   of un_op_expr      (* { op; arg } *)
  | Cond   of cond_expr       (* { pred; then_; else_ } *)
  | TimeFunc of string        (* school term forcing etc — constant in θ *)
  | TableLookup of string * expr list  (* covariate lookup — constant in θ *)
  | Projected                 (* observation projection — not in rates *)
```

### The `differentiate` function

Add to `ocaml/lib/ir/ir.ml` (or a new `ocaml/lib/ir/autodiff.ml`):

```ocaml
(** Symbolic differentiation of an expression with respect to a named parameter.
    Returns a new expr representing ∂expr/∂param.

    Key invariant: Pop, PopSum, Time, TimeFunc, TableLookup are all treated
    as constants (derivative = 0). In the PGAS θ|X step, compartment counts
    are fixed (conditioned on the trajectory X), time is fixed (iterating over
    known substeps), and covariates are data. Only Param nodes have nonzero
    derivatives. *)
let rec differentiate (e : expr) (param : string) : expr =
  match e with
  (* Constants — derivative is zero *)
  | Const _     -> Const 0.0
  | Pop _       -> Const 0.0
  | PopSum _    -> Const 0.0
  | Time        -> Const 0.0
  | TimeFunc _  -> Const 0.0
  | TableLookup _ -> Const 0.0
  | Projected   -> Const 0.0

  (* Parameter reference — 1 if it's the target, 0 otherwise *)
  | Param p     -> if p = param then Const 1.0 else Const 0.0

  (* Binary operations — standard calculus rules *)
  | BinOp b -> begin match b.op with
    | Add -> BinOp { op = Add;
                     left = differentiate b.left param;
                     right = differentiate b.right param }
    | Sub -> BinOp { op = Sub;
                     left = differentiate b.left param;
                     right = differentiate b.right param }

    (* Product rule: d(fg) = f'g + fg' *)
    | Mul -> BinOp { op = Add;
                     left = BinOp { op = Mul;
                                    left = differentiate b.left param;
                                    right = b.right };
                     right = BinOp { op = Mul;
                                     left = b.left;
                                     right = differentiate b.right param } }

    (* Quotient rule: d(f/g) = (f'g - fg') / g² *)
    | Div -> BinOp { op = Div;
                     left = BinOp { op = Sub;
                       left = BinOp { op = Mul;
                                      left = differentiate b.left param;
                                      right = b.right };
                       right = BinOp { op = Mul;
                                       left = b.left;
                                       right = differentiate b.right param } };
                     right = BinOp { op = Mul;
                                     left = b.right;
                                     right = b.right } }

    (* Power rule: d(f^g)/dθ
       General case: f^g = exp(g * ln(f))
       d/dθ = f^g * (g' * ln(f) + g * f'/f)
       Special case when g is constant in θ: f^g * g * f'/f *)
    | Pow ->
      let df = differentiate b.left param in
      let dg = differentiate b.right param in
      (* Full general form: *)
      BinOp { op = Mul;
              left = BinOp { op = Pow; left = b.left; right = b.right };
              right = BinOp { op = Add;
                left = BinOp { op = Mul;
                               left = dg;
                               right = UnOp { op = Log; arg = b.left } };
                right = BinOp { op = Mul;
                                left = b.right;
                                right = BinOp { op = Div;
                                                left = df;
                                                right = b.left } } } }

    (* Min/Max: use subgradient (derivative of the active branch) *)
    | Min -> Cond { pred = BinOp { op = Lt; left = b.left; right = b.right };
                    then_ = differentiate b.left param;
                    else_ = differentiate b.right param }
    | Max -> Cond { pred = BinOp { op = Gt; left = b.left; right = b.right };
                    then_ = differentiate b.left param;
                    else_ = differentiate b.right param }

    (* Mod: not differentiable in general. Return 0.
       If this appears in a rate expression, it's likely an error. *)
    | Mod -> Const 0.0

    (* Comparison ops: derivative is 0 (they return 0 or 1) *)
    | Eq | Neq | Lt | Gt | Le | Ge -> Const 0.0
    end

  (* Unary operations — chain rule *)
  | UnOp u -> begin match u.op with
    (* d/dθ exp(f) = exp(f) * f' *)
    | Exp -> BinOp { op = Mul;
                     left = UnOp { op = Exp; arg = u.arg };
                     right = differentiate u.arg param }

    (* d/dθ log(f) = f' / f *)
    | Log -> BinOp { op = Div;
                     left = differentiate u.arg param;
                     right = u.arg }

    (* d/dθ sqrt(f) = f' / (2 * sqrt(f)) *)
    | Sqrt -> BinOp { op = Div;
                      left = differentiate u.arg param;
                      right = BinOp { op = Mul;
                                      left = Const 2.0;
                                      right = UnOp { op = Sqrt; arg = u.arg } } }

    (* d/dθ (-f) = -f' *)
    | Neg -> UnOp { op = Neg; arg = differentiate u.arg param }

    (* d/dθ |f| = f' * sign(f) = f' * f / |f| *)
    | Abs -> BinOp { op = Mul;
                     left = differentiate u.arg param;
                     right = BinOp { op = Div;
                                     left = u.arg;
                                     right = UnOp { op = Abs; arg = u.arg } } }

    (* Floor/Ceil: piecewise constant, derivative = 0 *)
    | Floor -> Const 0.0
    | Ceil  -> Const 0.0
    end

  (* Conditional: differentiate both branches *)
  | Cond c -> Cond { pred = c.pred;  (* don't differentiate the predicate *)
                     then_ = differentiate c.then_ param;
                     else_ = differentiate c.else_ param }
```

### Expression simplification

The raw derivative expressions contain many redundant terms (multiply by 0,
add 0, multiply by 1). Add a simplifier to reduce expression size before
serialization:

```ocaml
(** Algebraic simplification: constant folding, identity elimination.
    Apply repeatedly until fixed point. *)
let rec simplify (e : expr) : expr =
  match e with
  (* 0 + x = x, x + 0 = x *)
  | BinOp { op = Add; left = Const 0.0; right = x }
  | BinOp { op = Add; left = x; right = Const 0.0 } -> simplify x

  (* x - 0 = x *)
  | BinOp { op = Sub; left = x; right = Const 0.0 } -> simplify x

  (* 0 * x = 0, x * 0 = 0 *)
  | BinOp { op = Mul; left = Const 0.0; _ }
  | BinOp { op = Mul; right = Const 0.0; _ } -> Const 0.0

  (* 1 * x = x, x * 1 = x *)
  | BinOp { op = Mul; left = Const 1.0; right = x }
  | BinOp { op = Mul; left = x; right = Const 1.0 } -> simplify x

  (* 0 / x = 0 *)
  | BinOp { op = Div; left = Const 0.0; _ } -> Const 0.0

  (* x / 1 = x *)
  | BinOp { op = Div; left = x; right = Const 1.0 } -> simplify x

  (* x ^ 0 = 1, x ^ 1 = x *)
  | BinOp { op = Pow; right = Const 0.0; _ } -> Const 1.0
  | BinOp { op = Pow; left = x; right = Const 1.0 } -> simplify x

  (* Constant folding: op(const, const) = const *)
  | BinOp { op; left = Const a; right = Const b } ->
    Const (eval_bin_op op a b)

  | UnOp { op; arg = Const a } ->
    Const (eval_un_op op a)

  (* -0 = 0 *)
  | UnOp { op = Neg; arg = Const 0.0 } -> Const 0.0

  (* Recurse into subexpressions *)
  | BinOp b -> let e' = BinOp { b with left = simplify b.left;
                                        right = simplify b.right } in
               if e' = e then e' else simplify e'
  | UnOp u  -> let e' = UnOp { u with arg = simplify u.arg } in
               if e' = e then e' else simplify e'
  | Cond c  -> Cond { pred = simplify c.pred;
                       then_ = simplify c.then_;
                       else_ = simplify c.else_ }
  | _ -> e
```

### Where derivatives are emitted

In the IR JSON, each transition already has a `rate` field containing an `expr`.
Add a new optional field `rate_grad` containing a map from parameter name to
the derivative expression:

```json
{
  "name": "infection",
  "rate": { "bin_op": { "op": "mul", "left": ..., "right": ... } },
  "rate_grad": {
    "R0": { "bin_op": { "op": "mul", "left": ..., "right": ... } },
    "gamma": { "const": 0.0 },
    "amplitude": { ... }
  }
}
```

The OCaml compiler populates `rate_grad` when an `[estimate]` section is present
in the model configuration (i.e., when gradient-based inference might be used).
If no `[estimate]` section, `rate_grad` is omitted (backward compatible).

Similarly, the overdispersion expression `sigma_sq` in `DrawOverdispersed(expr)`
gets a `sigma_sq_grad` map.

## Part 2: Rust Runtime — Gradient Evaluation

### No changes to `eval_expr`

The derivative expressions are plain `Expr` trees. Rust evaluates them with
the existing `eval_expr` function. No new evaluation machinery needed.

### New function: `log_transition_density_grad`

```rust
/// Evaluate the log transition density AND its gradient w.r.t. estimated params.
///
/// Returns (log_p, grad) where grad[i] = ∂log_p/∂θ_i.
///
/// The gradient is computed using the pre-differentiated rate expressions
/// emitted by the OCaml compiler (the `rate_grad` fields in the IR).
/// No runtime AD — just evaluating pre-computed derivative expressions.
pub fn log_transition_density_grad(
    model: &CompiledModel,
    counts_before: &[i64],
    flows: &[u64],
    gammas: &[f64],
    params: &[f64],
    t: f64,
    dt: f64,
    estimated_param_names: &[String],
) -> Result<(f64, Vec<f64>), SimError> {
    let d = estimated_param_names.len();
    let mut log_p = 0.0;
    let mut grad = vec![0.0; d];

    // ... for each source group, compute rates and derivative of rates ...

    // For each transition's rate expression:
    //   rate_val = eval_expr(&transition.rate, &ctx)
    //   rate_grad[i] = eval_expr(&transition.rate_grad[param_name], &ctx)

    // Chain rule through p_total = 1 - exp(-total_rate * dt):
    //   dp_total/dθ = dt * exp(-total_rate * dt) * d(total_rate)/dθ

    // Chain rule through binom_logpmf(k, n, p):
    //   d/dp [k*ln(p) + (n-k)*ln(1-p)] = k/p - (n-k)/(1-p)
    //   d/dθ binom_logpmf = (k/p - (n-k)/(1-p)) * dp/dθ

    // Chain rule through gamma_logpdf(g, shape, scale):
    //   d/d(σ²) log Gamma(g; dt/σ², σ²/dt)
    //   = d/d(σ²) [(dt/σ² - 1)*ln(g) - g*dt/σ² - dt/σ²*ln(σ²/dt) - lgamma(dt/σ²)]

    Ok((log_p, grad))
}
```

The chain rule through `p_total` and `binom_logpmf` is hardcoded in Rust (not
in the OCaml compiler) because these are inference-specific computations, not
model expressions. The OCaml compiler only differentiates rate expressions
and overdispersion expressions — the parts that come from the `.camdl` file.

### New function: `complete_data_loglik_grad`

```rust
/// Gradient of the complete-data log-likelihood over all substeps.
///
/// Sums log_transition_density_grad over all substeps and
/// dmeasure_grad over all observation times.
pub fn complete_data_loglik_grad(
    model: &CompiledModel,
    trajectory: &PGASTrajectory,
    observations: &[Observation],
    params: &[f64],
    estimated_params: &[String],
    dmeasure_fn: &DmeasureFn,
    flow_indices: &[usize],
    dt: f64,
) -> (f64, Vec<f64>) {
    let mut total_ll = 0.0;
    let mut total_grad = vec![0.0; estimated_params.len()];

    // Sum over substeps
    for s in 0..trajectory.substeps.len() {
        let counts_before = if s == 0 {
            &trajectory.initial_counts
        } else {
            &trajectory.substeps[s - 1].counts
        };
        let (ll_s, grad_s) = log_transition_density_grad(
            model, counts_before, &trajectory.substeps[s].flows,
            &trajectory.substeps[s].gammas, params, t_of_substep(s), dt,
            estimated_params,
        ).unwrap_or((f64::NEG_INFINITY, vec![0.0; estimated_params.len()]));
        total_ll += ll_s;
        for i in 0..total_grad.len() { total_grad[i] += grad_s[i]; }
    }

    // Sum over observations (dmeasure gradient)
    // For now: dmeasure gradient can be computed by finite differences
    // (1 PF eval per param, cheap since dmeasure is fast).
    // Future: differentiate the observation model expressions too.
    for obs in observations { /* ... */ }

    (total_ll, total_grad)
}
```

## Part 3: NUTS Integration

NUTS implementation is ~200 lines (leapfrog + U-turn + tree doubling).
Reference: Algorithm 6 in Hoffman & Gelman (2014), "The No-U-Turn Sampler."

### Expected performance

One gradient evaluation: ~46K substeps × eval_expr on ~6 derivative expressions
each ≈ 10–20ms. NUTS with ~10 leapfrog steps ≈ 100–200ms per θ|X update.
Plus CSMC-AS ≈ 500ms. Total ≈ 700ms per sweep.

Compare to current: 0.5 sec/sweep but with ESS/sweep ≈ 0.05–0.1.
With NUTS: 0.7 sec/sweep but with ESS/sweep ≈ 0.5–1.0.
**Net: 5–20× improvement in ESS per wall-clock second.**

## Part 4: dmeasure Gradients

The observation model also contributes to the complete-data log-likelihood.
For He et al. (discretized Normal):

```
log p(y | x, θ) = log Normal(y; ρ·H, ρ(1-ρ)H + ψ²H²)
```

With ρ and ψ fixed, this has no θ-dependence and the gradient is zero.
If ρ or ψ are estimated, the derivatives are straightforward and can be
handled the same way: differentiate the observation model expressions in
the OCaml compiler, emit derivative expressions in the IR.

For the initial implementation, assume observation model parameters are fixed
(they are in the He et al. analysis). Add observation model differentiation
as a follow-up.

### Structural insight: coordinate system safety

The double chain rule bug (#1) came from mixing gradients in different
coordinate systems (d/dθ vs d/dz) without type distinction — all `f64`.
The fix follows Stan's principle: **one function, one scale, one gradient
vector.** Each component (LL, prior, Jacobian) returns `(value, d/dz)`.
The caller just sums. No cross-scale mixing, no manual chain rule at the
call site.

## Phase 5: Closing the Gradient Gaps

Two gradient terms are missing from `pgas_grad.rs`, preventing
PGAS+NUTS from estimating:

1. **sigma_se** (overdispersion parameter) — needs the gamma
   density gradient
2. **Observation model parameters** (rho, psi, k) — needs
   d(dmeasure)/d(param)

Both require ~550 lines total. No architecture changes. No OCaml
compiler changes.

### 5.1: Digamma function

Both gradients need ψ(x) = d/dx ln Γ(x). Asymptotic expansion for
x > 6, recurrence ψ(x+1) = ψ(x) + 1/x for smaller x. Also
`log_gamma_density(x, shape, scale)` for the gamma multiplier density.
Add to `obs_loglik.rs` (~40 lines + tests).

### 5.2: Runtime expression differentiation

Rather than extending the OCaml compiler, implement forward-mode AD
on the Expr tree at runtime: `eval_expr_deriv(expr, wrt_param_idx, ctx)`.
Walks the AST with standard rules (product, quotient, chain). All
constant-in-θ nodes (Pop, Time, TimeFunc, TableLookup) return 0.
Add to `propensity.rs` (~80 lines + tests).

This is needed for sigma_sq and observation model expressions that
weren't covered by the compile-time `rate_grad` pass.

### 5.3: Gamma density gradient for sigma_se

When sigma_se is estimated, each gamma multiplier g_s was drawn from
Gamma(dt/σ², σ²/dt). The complete-data LL must include:

```
log p(g | σ²) = Σ_s log Gamma(g_s; dt/σ², σ²/dt)
```

The gradient w.r.t. σ² uses digamma:

```
d/d(σ²) = (-dt/σ⁴) × [ln(g) - ln(σ²/dt) - ψ(dt/σ²)]
         + (1/dt) × [g·dt²/σ⁴ - dt/σ²·dt/σ²]
```

Chain rule through the sigma_sq expression via `eval_expr_deriv`.
Add `log_gamma_density_grad_substep` to `pgas_grad.rs` (~60 lines).

### 5.4: Per-family observation model gradients

Add gradient functions for each likelihood family:

- `negbin_logpmf_grad(y, mu, k) → (d_mu, d_k)` — uses digamma
- `discretized_normal_logpmf_grad(y, mu, var, tol) → (d_mu, d_var)`
- `normal_logpdf_grad(y, mu, sigma) → (d_mu, d_sigma)`
- `poisson_logpmf_grad(k, lambda) → d_lambda`

Add to `obs_loglik.rs` (~80 lines + finite-diff tests).

### 5.5: Observation model gradient closure

`compile_dmeasure_grad` builds a closure from the IR observation model
that returns `(loglik, d_loglik/d_params)`. Uses `eval_expr_deriv` to
chain-rule through the mean/sd/dispersion expressions. Add to
`dmeasure.rs` (~80 lines).

### 5.6: Wiring into complete_data_loglik_grad

- Add gamma density gradient at each substep
- Add observation model gradient at observation times
- Pass optional `DmeasureGradFn` to `complete_data_loglik_grad`
