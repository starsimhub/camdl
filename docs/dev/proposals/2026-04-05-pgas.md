# Proposal: PGAS (Particle Gibbs with Ancestor Sampling)
**Motivation:** CPM-PMMH hits a ceiling of ρ_eff ≈ 0.68 due to discrete-state
dynamics breaking correlation through resampling. PGAS avoids PF variance
entirely by conditioning on a reference trajectory — no likelihood ratio,
no correlation needed.

**Reference:** Lindsten, Jordan & Schön (2014), "Particle Gibbs with ancestor
sampling." JMLR 15:2145–2184.

## Algorithm Overview

PGAS is a Gibbs sampler alternating between:

1. **θ | X, y** — update parameters given the full latent trajectory
2. **X | θ, y** — update the latent trajectory given parameters

Step 2 uses Conditional SMC with Ancestor Sampling (CSMC-AS), which avoids
the PF variance problem that plagues PMMH.

### One Gibbs Sweep

```
Input: current (θ, X_{1:T}), observations y_{1:T}
Output: new (θ', X'_{1:T})

// Step 1: update parameters
for each parameter group g:
    propose θ'_g from q(·| θ_g)
    log α = log p(y, X | θ') + log p(θ') - log p(y, X | θ) - log p(θ)
    // ↑ This is EXACT — no PF needed, X is known
    accept/reject via MH

// Step 2: update trajectory
X'_{1:T} = CSMC_AS(θ', y_{1:T}, X_{1:T})
// ↑ Returns an exact draw from p(X | θ', y)
```

### Why Step 1 is Exact

With the full trajectory X known, the complete-data log-likelihood is:

```
log p(y, X | θ) = Σ_t log p(y_t | x_t, θ)           // observation
               + Σ_t log p(x_t | x_{t-1}, θ)        // transition
               + log p(x_0 | θ)                       // initial
```

Every term is a known density evaluated at known states. No particle filter,
no estimation noise, no variance scaling with T. The MH ratio for parameter
updates is as clean as in any standard MCMC.

### CSMC-AS (Conditional SMC with Ancestor Sampling)

This is a modified particle filter where one particle slot (say slot J) is
clamped to the reference trajectory X from the previous Gibbs sweep.

```
for t = 1 to T:
    // Propagate free particles (slots 1..J-1)
    for j = 1 to J-1:
        x_t^j = simulate(x_{t-1}^{a_j}, θ, t)   // standard PF

    // Clamp reference particle (slot J)
    x_t^J = X_t   // the reference trajectory value

    // Compute weights
    for j = 1 to J:
        w_t^j = p(y_t | x_t^j, θ)   // observation density (dmeasure)

    // ── ANCESTOR SAMPLING (the key innovation) ──
    // For the reference particle, choose its ancestor by evaluating
    // the transition density from each particle to the reference's
    // NEXT state:
    for j = 1 to J:
        ã_t^j = w_{t-1}^j · p(X_{t+1} | x_t^j, θ)
    a_J = sample(categorical(ã))
    // This allows the reference to "reconnect" to any particle's
    // history, maintaining diversity across sweeps.

    // Resample free particles (standard systematic resampling)
    resample slots 1..J-1 from weights w_t

// At the end: sample one trajectory from final weights
k ~ categorical(w_T)
X'_{1:T} = ancestry of particle k
```

The ancestor sampling step is what makes PGAS work for long time series.
Without it, the reference trajectory dominates and the sampler degenerates
(path degeneracy). With it, the reference can be "spliced" with better
particle histories at each observation time.

## Transition Density: The Hard Part

Ancestor sampling requires evaluating `p(x_{t+1} | x_t, θ)` — the probability
that the chain binomial, starting from state x_t with parameters θ, produces
exactly state x_{t+1} in one observation interval (7 daily substeps for weekly
measles data).

### Structure of the chain binomial transition

For one daily substep, starting from counts = (S, E, I, R):

**Source group: S** (transitions: infection S→E, death S→∅)
1. Compute per-capita rates: r_inf = β(t)·(I+ι)^α/N·g, r_death = μ
   where g ~ Gamma(dt/σ_SE², σ_SE²/dt) is the noise multiplier
2. Total exit prob: p_total = 1 - exp(-(r_inf + r_death)·dt)
3. Total exits: n_exit ~ Binom(S, p_total)
4. Split: n_inf ~ Binom(n_exit, r_inf/(r_inf + r_death)), n_death = n_exit - n_inf

**Source group: E** (transitions: progression E→I, death E→∅)
1. Per-capita rates: r_prog = σ, r_death = μ
2. Total exit prob: p_total = 1 - exp(-(σ + μ)·dt)
3. Total exits: n_exit ~ Binom(E, p_total)
4. Split: n_prog ~ Binom(n_exit, σ/(σ+μ)), n_death = n_exit - n_prog

**Source group: I** (transitions: recovery I→R, death I→∅)
1. Per-capita rates: r_rec = γ, r_death = μ
2. Total exit prob: p_total = 1 - exp(-(γ + μ)·dt)
3. Total exits: n_exit ~ Binom(I, p_total)
4. Split: n_rec ~ Binom(n_exit, γ/(γ+μ)), n_death = n_exit - n_rec

**Births:** deterministic (round(birthrate·pop·dt)), no density needed.

### The transition density for one substep

Given (x_prev, x_next, flows, gamma_multiplier), the log-probability is:

```
log p(x_next | x_prev, θ, g) =
    // S source group
    log Binom(n_exit_S; S_prev, p_total_S)
    + log Binom(n_inf; n_exit_S, r_inf/(r_inf + r_death_S))
    // E source group
    + log Binom(n_exit_E; E_prev, p_total_E)
    + log Binom(n_prog; n_exit_E, σ/(σ+μ))
    // I source group
    + log Binom(n_exit_I; I_prev, p_total_I)
    + log Binom(n_rec; n_exit_I, γ/(γ+μ))
    // Gamma multiplier density
    + log Gamma(g; dt/σ², σ²/dt)
```

where transition counts are recovered from per-substep flow accumulators.

### Critical implementation note

**The transition density must exactly mirror `step_one`'s decomposition.**
The rates, the order of sequential conditional binomials, the handling of
the last category as remainder — all must match. If the density function
computes `p_split` differently from how `step_one` drew the split, the
ancestor weights will be wrong and the sampler will degenerate silently
(it'll run, but mixing will be terrible).

**Recommended implementation approach:** Copy `step_one` and replace every
`rng.binomial(n, p)` with `+= binom_logpmf(observed_count, n, p)`.
Same rates, same ordering, same conditional structure. The density
function IS `step_one` with draws replaced by probability evaluations.

### The Gamma multiplier

The overdispersed infection rate uses g ~ Gamma(dt/σ_SE², σ_SE²/dt). We
use **Option B: Augment the state with g.** Store the Gamma multiplier as
part of the trajectory at each substep. The transition density conditions
on g (known) and is just the product of Binomials. The Gibbs sampler gets
an additional step: sample g | X, θ, y, which is conjugate
(Gamma posterior from Gamma prior × Binomial likelihood).

Storage cost: 7672 substeps × 8 bytes = 61 KB per trajectory. Trivial.

### Multi-substep transition: daily resampling

**Daily resampling is non-negotiable.** Resampling weekly and somehow
handling the 7-substep transition density would require summing over
intermediate states (intractable). Daily resampling with trivial weights
(all 1.0) on non-observation days is clean and correct:

- 7672 resampling steps (daily) instead of 1096 (weekly)
- At non-observation days: all weights = 1 (uniform), resampling is a no-op
- At observation days: weights = p(y_t | x_t, θ) via dmeasure
- Ancestor sampling at every daily step using the single-substep transition
  density (tractable)

## Configuration

```toml
[pgas]
chains = 4
sweeps = 10000
particles = 100       # PGAS needs far fewer particles than PMMH
burn_in = 2000
thin = 5
```

PGAS typically needs J=50–200 particles, vs J=2000+ for PMMH.

## Expected performance

Per sweep: 100 particles × 7672 substeps × ~1μs/step ≈ 0.77 sec for CSMC,
plus ~10ms for parameter updates. Total: ~0.8 sec/sweep.

10K sweeps × 0.8 sec = ~2.2 hours per chain. 4 chains parallel ≈ 2.2 hours
on M4 Max.

Compare PMMH: 2000 steps × 7.5 sec/step = 4.2 hours, 5% acceptance → ~100
effective samples. PGAS should produce ~2000+ effective samples in half the time.

## Risk assessment

**Main risk: path degeneracy despite ancestor sampling.** If observation noise
is very small (tight ψ), ancestor weights concentrate on one particle and
the reference trajectory is rarely updated. Mitigation: check conditional ESS
diagnostic. If < 10% of steps get a new ancestor, increase J.

**Secondary risk: transition density numerical issues.** Very large
compartments (S ≈ 3M) produce Binom pmfs with extreme n. Need log-space
Stirling's approximation via existing `lgamma` in `obs_loglik.rs`.

**Low risk: parameter update acceptance.** With X fixed, the posterior over
θ is smooth. 1D MH updates with adaptive step sizes should get 30-50%
acceptance.

## Testing strategy

**Test on the pure death model first.** It has an analytical posterior, so
you can verify PGAS is sampling correctly before touching measles. If the
pure death posterior doesn't match the analytical answer, the transition
density function has a bug. Don't skip this step.

1. **Pure death model** (analytical posterior): verify posterior mean and
   variance match within Monte Carlo error.
2. **SIR with known parameters**: verify sampled trajectories bracket truth.
3. **He et al. 3-param** (R0, amplitude, s0): compare to PMMH point
   estimates.
4. **He et al. 5-param**: production test case.
